# mdtool-chrome — итоговый план и roadmap

Версия: 1.0  
Фокус: универсальный рабочий клиппер + экспорт Confluence **без OAuth и без API-ключей**.
Статус реализации: этапы A/B/C/D реализованы в коде, критерии качества подтверждаются отдельными E2E-прогонами.

---

## 1) Продуктовая цель

Сделать бесплатное расширение, которое:

1. Стабильно сохраняет контент большинства веб-страниц в Markdown.
2. Умеет выгружать Confluence Space в структуру папок/файлов.
3. Работает без отдельной авторизации в расширении и без токенов/API-ключей.

---

## 2) Ограничение по Confluence (принятое решение)

### Что выбираем

- **Только session-based mode**: расширение использует уже активную сессию пользователя в Confluence (если пользователь уже залогинен в браузере).
- **Без OAuth**, **без API tokens**, **без ввода email/password** в расширении.

### Что это значит на практике

- Пользователь открывает Confluence в браузере и авторизуется обычным способом.
- Расширение ходит по страницам Space и извлекает контент как “пользователь в браузере”.
- Масштабируемость ограничена (это ок для MVP по текущему требованию).

### Ограничения режима

- Экспорт работает только в рамках прав текущего залогиненного пользователя.
- Медленнее, чем API-режим.
- Могут быть пропуски страниц/вложений на нестандартных/кастомных инстансах.
- Для очень больших Space возможны таймауты/прерывания.

---

## 3) Целевое состояние (Definition of Done)

### Web Clipper

- Popup extraction стабилен на типовых сайтах документации.
- Работают режимы: article/full/selection/code/tables.
- Сохранение в историю + скачивание `.md` + batch `.zip`.
- Ошибки понятны пользователю (не “пусто”, а причина).

### Confluence (No-auth MVP)

- Пользователь вводит URL Space (или key) и запускает экспорт.
- На выходе zip:
  - `confluence/<SPACE>/.../*.md`
  - `confluence/<SPACE>/_attachments/*`
- Для каждой страницы: frontmatter + markdown.
- Прогресс и итоговый отчёт: `exported / skipped / failed`.

## Этап C — Confluence Space Export без авторизации (MVP) (2-4 недели)

### Архитектурный подход (без API ключей)

- Расширение работает только если открыт/доступен Confluence домен в текущей сессии браузера.
- Источник структуры:
  - UI-навигация Space (sidebar / pages list),
  - обход найденных ссылок (BFS очередь).
- Для каждой страницы:
  - открыть страницу (в фоне/по очереди),
  - извлечь основной контент,
  - конвертировать в markdown,
  - сохранить метаданные (title/url/breadcrumb/path).

### Задачи

  - [x] поле `Space URL or key`,
  - [x] кнопка `Scan`,
  - [x] список найденных страниц,
  - [x] кнопка `Export`.
- [x] Crawler:
  - [x] очередь ссылок,
  - [x] дедупликация URL,
  - [x] ограничение глубины/количества,
  - [x] стоп,
  - [x] пауза.
- [x] Построение структуры директорий:
  - [x] через breadcrumbs/иерархию URL,
  - [x] fallback по title slug.
- [x] Вложения:
  - [x] попытка скачать файлы/изображения страницы,
  - [x] rewrite ссылок в markdown на локальные пути.
- [x] Итоговый zip + отчёт.

### Критерий готовности

- [ ] Экспорт small/medium Space (до ~200 страниц) с приемлемым качеством.
- Примечание: реализован рабочий no-auth MVP (scan/export/zip/report), требуется отдельный end-to-end прогон на реальном Space.

## 5) Функциональный scope (что входит/не входит)

### Входит в MVP

- Универсальный Web->Markdown клиппер.
- Batch export вкладок.
- Confluence Space export через активную сессию браузера.

### Не входит в MVP (позже)

- OAuth/токены.
- Высокомасштабный server-side crawl.
- Идеальная поддержка всех Confluence макросов.

---

## 6) Ключевые риски и меры

- **UI Confluence может меняться** -> делать несколько стратегий поиска ссылок.
- **Большие Space** -> лимиты и чанкинг, опция “экспорт частями”.
- **Неполные права пользователя** -> явно показывать “частичный экспорт”.
- **Падения на сложных страницах** -> fallback extraction + отчёт ошибок.

---

## 7) Метрики успеха

- Успешность extraction web-страниц: >90%.
- Успешность экспорта Confluence страниц в Space: >85% для target-инстансов.
- Среднее время экспорта 100 страниц: в целевом SLA (фиксируется после первого прогона).
- Доля “тихих” ошибок в UI: ~0 (все ошибки видимы пользователю).

---

## 8) Приоритеты реализации

1. Надёжность текущего extraction/save.
2. Batch и zip без регрессий.
3. Confluence no-auth MVP (session crawl).
4. Hardening/quality/reporting.

---

## 9) Итоговая стратегия

Для текущего требования лучшее решение:

- Выпускаем рабочий бесплатный MVP **без авторизаций и API-ключей**.
- Опираться на текущую browser session пользователя.
- Жертвуем масштабируемостью и частью edge-case покрытия ради скорости вывода в прод.

Если позже потребуется enterprise-качество и масштаб, добавляем отдельный “advanced mode” с API/OAuth, не ломая no-auth путь.
