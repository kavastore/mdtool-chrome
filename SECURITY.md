# Security Policy

## Supported versions

Security fixes are applied to the latest release on `master`.

## Reporting a vulnerability

If you discover a security issue, please open a private report first:

- GitHub security advisory (preferred), or
- direct contact via repository owner channels.

Please include:

- affected version/commit,
- impact summary,
- reproduction steps or proof of concept,
- suggested mitigation (if available).

We will acknowledge reports as quickly as possible and coordinate responsible disclosure.
